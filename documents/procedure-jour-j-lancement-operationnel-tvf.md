# Procedure jour J - lancement operationnel TVF

Document interne pour organiser le premier jour d'utilisation reelle du site, de TVF OS, des formulaires, de la boite e-mail officielle et de TVF Mobile.

## Objectif

Passer d'un environnement pret techniquement a une exploitation simple, controlee et tracable. Le jour J doit permettre de verifier les vrais flux sans melanger les tests, les demandes utiles et les actions de prospection.

## Organisation du traitement

### 1. Reception

Toute demande doit arriver dans une seule logique de traitement :

- formulaire du site ;
- demande TVF Mobile ;
- e-mail recu a l'adresse officielle ;
- contact direct lors d'un rendez-vous ;
- contact issu d'une prospection.

Si la demande ne vient pas automatiquement du site ou de TVF Mobile, elle doit etre creee manuellement dans TVF OS.

### 2. Qualification

Chaque demande doit recevoir au minimum :

- un type d'interlocuteur ;
- une categorie ;
- une commune ;
- un niveau de priorite ;
- un statut ;
- une prochaine action.

### 3. Transformation en dossier

Une demande devient un dossier quand elle necessite :

- une analyse ;
- des pieces a fournir ;
- un rendez-vous ;
- une visite ;
- une convention ;
- une reponse structuree ;
- un suivi dans le temps.

### 4. Documents

Les documents internes ne doivent pas etre envoyes automatiquement. Ils sont choisis selon le besoin : collectivite, proprietaire, entreprise, materiaux, benevolat, financeur, insertion ou stockage.

### 5. Reponse

Toute reponse doit etre claire :

- rappeler la demande recue ;
- indiquer si TVF peut l'etudier ;
- preciser les pieces manquantes ;
- proposer une prochaine etape ;
- ne jamais promettre une renovation, un financement ou une acceptation automatique.

## Rythme de suivi les 7 premiers jours

| Jour | Priorite | Action |
|---|---|---|
| J1 | Reception | Controler tous les formulaires et la boite mail |
| J2 | Qualification | Classer toutes les demandes recues |
| J3 | Dossiers | Transformer les demandes utiles en dossiers |
| J4 | Documents | Envoyer les premieres listes de pieces ou courriers |
| J5 | Relances | Relancer les demandes incompletes |
| J6 | Analyse | Identifier les demandes a fort potentiel |
| J7 | Reporting | Faire une synthese : demandes, dossiers, blocages, suites |

## Indicateurs de demarrage

| Indicateur | Definition | Objectif de suivi |
|---|---|---|
| Demandes recues | Nombre de demandes entrantes | Comprendre le volume |
| Demandes qualifiees | Demandes avec categorie et priorite | Eviter les demandes floues |
| Dossiers crees | Demandes transformees en dossiers | Suivre les cas utiles |
| Pieces manquantes | Dossiers incomplets | Organiser les relances |
| Reponses envoyees | Demandes ayant recu une suite | Eviter les oublis |
| Rendez-vous prevus | Echanges a venir | Piloter le terrain |
| Incidents | Blocages techniques ou metier | Corriger rapidement |

## Feu vert jour J

Le lancement peut etre considere stable si :

- aucune demande n'est perdue ;
- les notifications arrivent correctement ;
- les demandes sont visibles dans TVF OS ;
- les dossiers peuvent etre crees ;
- les documents sont accessibles ;
- les reponses peuvent etre preparees ;
- les incidents sont notes dans un registre.
