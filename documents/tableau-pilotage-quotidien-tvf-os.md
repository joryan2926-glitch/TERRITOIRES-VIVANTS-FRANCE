# Tableau de pilotage quotidien TVF OS

Document interne pour ouvrir TVF OS chaque matin, traiter les priorites et garder une vision simple de l'activite.

## 1. Objectif

Le pilotage quotidien doit aider a repondre rapidement a quatre questions :

1. Quelles demandes doivent etre lues aujourd'hui ?
2. Quels dossiers attendent une action ?
3. Quelles pieces ou reponses manquent ?
4. Quelles relances doivent etre envoyees ?

Ce tableau ne remplace pas TVF OS. Il sert de grille de lecture rapide pour organiser la journee.

## 2. Routine du matin

| Ordre | Action | Module TVF OS | Resultat attendu |
|---:|---|---|---|
| 1 | Consulter les nouvelles demandes | Demandes | Identifier les demandes non lues |
| 2 | Verifier les demandes urgentes | Demandes | Prioriser securite, collectivite, local, financeur, gros volume |
| 3 | Controler les e-mails entrants | Boite mail | Repondre, convertir en demande ou classer |
| 4 | Ouvrir les dossiers actifs | Dossiers | Verifier la prochaine action |
| 5 | Lister les pieces manquantes | Documents | Preparer une relance |
| 6 | Verifier les taches du jour | Taches | Planifier appel, visite, courrier, rendez-vous |
| 7 | Tracer les decisions | Journal | Conserver la preuve de traitement |

## 8. Fin de journee

Avant de fermer TVF OS :

- [ ] Toutes les demandes nouvelles ont ete vues.
- [ ] Les demandes urgentes ont une prochaine action.
- [ ] Les e-mails importants ont ete traites ou convertis.
- [ ] Les dossiers actifs ont une date de prochaine action.
- [ ] Les pieces manquantes sont listees.
- [ ] Les relances du jour sont tracees.
- [ ] Les decisions importantes sont inscrites dans le journal.

## 9. Regle interne

Une demande sans prochaine action devient vite invisible. Chaque demande active doit avoir au minimum :

- un statut ;
- une categorie ;
- une priorite ;
- un interlocuteur ;
- une prochaine action ;
- une date de suivi.
