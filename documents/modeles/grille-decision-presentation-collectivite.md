# Grille de decision - Avant presentation a une collectivite

Statut : outil interne TVF.
Objectif : verifier qu'un dossier est presentable sans fragiliser la credibilite de l'association.

## 1. Niveau de maturite du dossier

| Niveau | Description | Peut etre presente ? | Condition |
| --- | --- | --- | --- |
| Idee | Besoin general, sans bien ni donnees qualifiees | Oui, comme intention | Dire clairement que c'est exploratoire |
| Signalement | Adresse, photo ou ressource transmise | Oui, avec prudence | Ne pas publier l'adresse sensible sans cadre |
| Pre-qualification | Besoin, proprietaire potentiel, dispositif compatible, risques identifies | Oui | Demander un accord pour instruire |
| Projet a instruire | Fiche projet, budget preliminaire, acteurs a mobiliser | Oui | Distinguer confirme et a rechercher |
| Projet conventionnable | Accord de principe, trame de convention, budget et responsabilites clarifies | Oui | Prevoir validation juridique/technique |
| Projet valide | Convention signee et conditions reunies | Oui | Publier seulement les informations autorisees |

## 2. Controle de credibilite

- [ ] Le besoin public est clairement formule.
- [ ] Les donnees utilisees sont sourcees ou presentees comme a verifier.
- [ ] Les proprietaires ou ayants droit ne sont pas supposes sans preuve.
- [ ] Les partenaires ne sont pas nommes sans accord.
- [ ] Les financeurs ne sont pas annonces sans notification ou engagement ecrit.
- [ ] Les montants sont distingues : estimation, devis, enveloppe recherchee, financement confirme.
- [ ] Les risques sont cites : juridique, technique, securite, assurance, urbanisme, donnees.
- [ ] Le statut du projet est visible : idee, etude, instruction, convention, realise.
- [ ] La collectivite comprend ce qui lui est demande.

## 3. Pieces a joindre si disponibles

- Fiche de reperage.
- Photos autorisees.
- Donnees publiques et sources.
- Carte ou croquis de localisation non sensible.
- Fiche budget et financement.
- Matrice besoin / dispositif / reponse TVF.
- Trame de convention adaptee.
- Checklist des points a verrouiller.
- Liste des decisions attendues.

## 4. Decision interne avant envoi

| Question | Oui / Non | Commentaire |
| --- | --- | --- |
| Le dossier est-il utile pour la collectivite ? |  |  |
| Le risque de surpromesse est-il maitrise ? |  |  |
| Les informations sensibles sont-elles protegees ? |  |  |
| Les prochaines etapes sont-elles simples ? |  |  |
| Une personne TVF peut-elle expliquer le dossier en 5 minutes ? |  |  |

## 5. Formulation recommandee

Utiliser des formulations prudentes :

- "TVF propose d'etudier..."
- "Une premiere phase pourrait consister a..."
- "Les donnees devront etre confirmees avec..."
- "Le financement reste a documenter..."
- "La convention devra preciser..."
- "Aucun engagement operationnel ne sera pris sans validation..."

Eviter :

- "Le projet est finance" sans preuve.
- "La collectivite est partenaire" sans accord.
- "Le chantier va demarrer" sans convention.
- "Les impacts seront de..." sans methode et donnees.
